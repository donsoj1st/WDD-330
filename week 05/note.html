NOTE

A note from the book: "Programmer errors are our responsibility, so we must ensure they are minimized as much as possible and fixed promptly. We also should try to limit user errors by predicting any possible input that the user might enter, that will lead to errors, in fact, also programmer errors, because the program should be designed in a way that prevents the user from making the error." I like the point the author is making about user errors being programmer errors. A well-designed program should take the user into account and be easy to use.

Throwing Exceptions

  • THROW Use throw statement to handle your own exceptions. Best practice dictates throwing an error object, which is caught in a catch block.
  • Exception Handling use to handle exceptions by catching errors in the catch block.
  • try, catch, and finally Wrap code that can result in an exception in a try block.
